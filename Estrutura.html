Estrutura Completa do Projeto Flutter - Klube Cash (App Cliente)

Este site interativo permite visualizar a arquitetura e estrutura de pastas do projeto Flutter Klube Cash. Clique nos cabeçalhos para expandir ou recolher as seções.

Visão Geral da Arquitetura

►

O projeto seguirá o padrão Clean Architecture com separação clara entre camadas de apresentação, domínio e dados. Utilizará Riverpod para gerenciamento de estado e Go Router para navegação.

Detalhamento dos Arquivos

►

Core

►

Constants:

  • api_constants.dart: URLs base da API, endpoints, timeouts
  • app_colors.dart: Paleta de cores (primária, secundária, success, error, etc)
  • app_images.dart: Caminhos das imagens e ícones
  • app_strings.dart: Todos os textos da aplicação (internacionalização)
  • app_dimensions.dart: Espaçamentos, tamanhos, bordas padrão

Errors:

  • exceptions.dart: Exceções customizadas (ServerException, CacheException)
  • failures.dart: Classes de falha para tratamento de erros

Network:

  • api_client.dart: Cliente HTTP configurado com Dio
  • network_info.dart: Verificação de conectividade
  • api_interceptors.dart: Interceptadores para auth, logs, retry

Routes:

  • app_router.dart: Configuração do Go Router com todas as rotas
  • route_guards.dart: Guards para verificar autenticação

Theme:

  • app_theme.dart: Tema light/dark completo
  • text_styles.dart: Estilos de texto padronizados
  • theme_provider.dart: Provider para alternar temas

Utils:

  • validators.dart: Validações de CPF, email, senha
  • formatters.dart: Formatação de moeda, data, CPF
  • date_utils.dart: Manipulação de datas
  • currency_utils.dart: Conversões e formatações monetárias

Widgets Globais:

  • custom_button.dart: Botão padrão com variações (primary, secondary, outline)
  • custom_text_field.dart: Campo de texto customizado com validação
  • loading_indicator.dart: Indicador de carregamento padrão
  • error_widget.dart: Widget para exibir erros
  • empty_state_widget.dart: Estado vazio customizável
  • custom_app_bar.dart: AppBar padrão do app

Features

►

Auth (Autenticação)

Screens:
  • splash_screen.dart: Logo animado do Klube Cash, Verificação de token salvo, Redirecionamento automático
  • login_screen.dart: Logo do Klube Cash no topo, Campo de email/CPF, Campo de senha com toggle de visibilidade, Checkbox "Lembrar-me", Botão "Entrar", Link "Esqueceu sua senha?", Divisor "ou", Botões de login social (Google, Facebook), Link "Não tem conta? Cadastre-se"
  • register_screen.dart: Header com voltar, Título "Criar Conta", Campo nome completo, Campo CPF com máscara, Campo email, Campo telefone com máscara, Campo senha com indicador de força, Campo confirmar senha, Checkbox de termos de uso, Botão "Cadastrar", Link "Já tem conta? Faça login"
  • recover_password_screen.dart: Header com voltar, Título "Recuperar Senha", Instrução explicativa, Campo email/CPF, Botão "Enviar", Mensagem de sucesso após envio

Dashboard

Screen:
  • dashboard_screen.dart: Header (Saudação personalizada, Foto do usuário, Ícone de notificações com badge), Card de Saldo Principal (Título "Seu Cashback", Saldo disponível em destaque, Saldo pendente menor abaixo, Toggle para ocultar valores, Botão "Ver extrato"), Gráfico de Evolução (Gráfico de linha dos últimos 30 dias, Seletor de período), Ações Rápidas (Grid 2x2: Ver lojas parceiras, Histórico completo, Indique amigos, Ajuda), Últimas Transações (Lista das 5 últimas, Logo da loja, Nome da loja, Data, Valor do cashback, Status, Link "Ver todas")

Cashback

Screens:
  • cashback_history_screen.dart: Header (Título "Histórico de Cashback", Botão de filtro), Card de Resumo (Total acumulado, Total disponível, Total pendente), Filtros (Bottom Sheet: Período, Status, Lojas específicas, Ordenação), Lista de Transações (Agrupadas por mês, Item: logo loja, nome, data, valor, status, Pull to refresh, Paginação infinita)
  • cashback_details_screen.dart: Logo e nome da loja, Status da transação, Valor da compra, Percentual de cashback, Valor do cashback, Data da compra, Previsão de liberação, Código da transação, Botão "Reportar problema"

Stores (Lojas Parceiras)

Screens:
  • partner_stores_screen.dart: Barra de Busca (Campo de pesquisa, Ícone de busca, Filtro por categoria), Chips de Categorias (Todas, Alimentação, Moda, Serviços, Outros), Lista/Grid de Lojas (Toggle lista/grid, Card: logo, nome, cashback%, categoria, Indicador de "Nova parceira"), Filtros Avançados (Por percentual de cashback, Por distância, Por avaliação)
  • store_details_screen.dart: Header com imagem (Foto de capa, Logo sobreposto, Botão voltar, Botão favoritar), Informações Principais (Nome da loja, Categoria, Percentual de cashback em destaque, Avaliação e reviews), Seções em tabs (Sobre, Localização, Avaliações), Botão Flutuante ("Ir às compras")

Profile

Screens:
  • profile_screen.dart: Header do Perfil (Foto, Nome, CPF parcial, Email, Membro desde), Menu de Opções (Meus dados, Alterar senha, Notificações, Termos de uso, Política de privacidade, Central de ajuda, Sobre o app, Sair), Card de Indicação ("Indique e ganha", Código de indicação, Botão compartilhar)
  • edit_profile_screen.dart: Foto editável, Campos: nome, email, telefone, Botão salvar, Validações em tempo real
  • change_password_screen.dart: Campo senha atual, Campo nova senha, Campo confirmar senha, Indicador de força, Botão salvar

Notifications

Screen:
  • notifications_screen.dart: Header (Título "Notificações", Botão "Marcar todas como lidas"), Lista de Notificações (Agrupadas por data, Ícone por tipo, Título e descrição, Horário, Indicador de não lida, Swipe para deletar)

Providers (Gerenciamento de Estado)

►
  • auth_provider.dart: Estado de autenticação (loading, authenticated, unauthenticated), Dados do usuário logado, Métodos: login, logout, register, checkAuth
  • dashboard_provider.dart: Saldo de cashback (disponível/pendente), Transações recentes, Dados do gráfico, Método refresh
  • cashback_provider.dart: Lista de transações de cashback, Filtros ativos, Paginação, Métodos: loadMore, filter, refresh
  • stores_provider.dart: Lista de lojas parceiras, Categorias, Filtros e busca, Lojas favoritas
  • profile_provider.dart: Dados do perfil, Estado de edição, Upload de foto
  • notifications_provider.dart: Lista de notificações, Contador de não lidas, Configurações de notificação

Fluxos Principais

►

Fluxo de Login:

  • Splash → Verifica token
  • Se não autenticado → Login
  • Validação de campos
  • Chamada API
  • Salva token
  • Navega para Dashboard

Fluxo de Cashback:

  • Dashboard mostra resumo
  • Cliente acessa histórico
  • Filtra por período/status
  • Visualiza detalhes
  • Acompanha liberação

Fluxo de Navegação Principal:

  • Bottom Navigation com 4 abas: Home (Dashboard), Histórico, Lojas, Perfil

Considerações de UX/UI

►

Cores principais:

  • Primária: Verde (cashback/dinheiro)
  • Secundária: Azul escuro
  • Success: Verde claro
  • Warning: Amarelo
  • Error: Vermelho

Animações:

  • Transições suaves entre telas
  • Shimmer durante carregamento
  • Pull to refresh animado
  • Animações de entrada de cards

Feedback visual:

  • Loading states
  • Empty states ilustrados
  • Mensagens de erro claras
  • Toasts para ações

Acessibilidade:

  • Textos com contraste adequado
  • Áreas de toque mínimas de 48px
  • Suporte a screen readers
  • Fontes escaláveis

Esta estrutura proporciona uma base sólida e escalável para o desenvolvimento do app cliente do Klube Cash, seguindo as melhores práticas do Flutter e garantindo uma experiência consistente e intuitiva para o usuário.
